Frequently asked questions

Clear answers before you add another tool.

Codex Coordinator is useful for a specific kind of problem. If your work is simpler, your workflow should stay simpler too.

What problem does it solve?

It helps when several Codex tasks are working in one repository and you are spending time checking windows, relaying updates, or untangling overlapping work.

When should I use it?

Use it for a larger repository goal that genuinely benefits from two or three durable tasks working at the same time. For one coherent job or a small edit, stay with one Codex task.

Will it create lots of task windows?

No. It starts with one task, reuses a suitable related task when possible, and opens another only for a substantial job that benefits from its own context.

Does it keep watching every task?

No. There is no background manager. Ask the Coordinator when you want the current picture or need the finished work brought together.

Does it copy my chats?

No. Task conversations stay in Codex. Coordinator stores only the small amount of local information needed to show who owns what.

Does it create Git worktrees?

No. Coordinated tasks stay in the checkout you are already using, so they can share local settings and offline tools. Worktrees remain available when you specifically want file and branch isolation.

Does it force a pull-request workflow?

No. Direct commits and normal pushes remain available. Use pull requests only when you or the repository already want them.

What happens when a task pauses?

Its current job and ownership remain visible, so you can resume or reassign the work without copying the whole conversation into another system.

What if the plugin is broken?

Update or reinstall it through the normal plugin manager. The included health check reports compatibility problems but does not attempt a complicated self-repair.

What do I need?

OpenAI Codex with plugin support, Git, and Python 3.10 or newer. Codex Coordinator itself is free and open source.

Where are the technical details?

The developer page covers the data format, runtime limits, validation, installation behavior, and design history without putting those details in the way of new users.
