Frequently asked questions

What the board does—and does not do.

What does Codex Coordinator do?

When you explicitly ask for coordination, one task reuses suitable related local tasks before assigning two or three complete verticals. Each native task publishes only its active goal, planned paths, narrow exclusive actions, dependencies, and status.

Why remove the resident Coordinator?

We removed the heartbeat, polling, ledgers, and repeated status checks—not the useful Coordinator role. The goal-scoped Coordinator stays available on demand and reads current state only when invoked.

Does it create task windows?

Only when you explicitly ask the Coordinator to decompose a substantial goal and no suitable related local task can be reused. It assigns complete durable verticals, not tiny check or review windows.

Does Codex Coordinator create Git worktrees?

No. Coordinated tasks stay on the current shared checkout and branch so they see the same local settings and offline runners. Each task may commit only its reviewed exact files; no task owns Git for its whole lifetime.

Does it store chats or reasoning?

No. Native Codex remains the transcript authority. Unknown claim fields are rejected and records are capped at 4 KB.

Does it keep watching in the background?

No. Agents update active state only when work starts, changes boundary, blocks, or finishes. The Coordinator reads it when invoked; SessionStart launches nothing.

Is Mission Control required?

No. Mission Control is not shipped in the schema-2 base package. Any future observer must be a separate manual, read-only package limited to the public board.

Can Doctor repair the installation?

No. Doctor only checks package compatibility. Broken packages are updated or reinstalled through the normal plugin manager.

Can I install schema 2 now?

Yes. Install v0.4.0 through the Codex plugin marketplace source. Installation does not enable any repository automatically; v0.3.0 remains the older orchestration design.

What does it require?

Codex with plugin and hook support, Git, and an existing Python 3.10+ interpreter. The core has zero third-party runtime dependencies.
