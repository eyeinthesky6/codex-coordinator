Task boundaries, not orchestration

Keep parallel Codex tasks from crossing wires.

Each native task publishes only its bounded goal, paths, and exclusive actions. No permanent manager task. No heartbeat. No transcript mirror.

Unreleased realignment

The simpler schema-2 design is not released or enabled yet.

The source implementation is under validation. Stable v0.3.0 is preserved as the older orchestration release and should not be enabled by users seeking the new behavior. A new install command will appear only after migration, performance, and release approval are complete.

Small hot path

Native Codex does the work. The board only shows planned ownership.

Retained protectionSimplified mechanism

No message loopsBoard first; sparse non-executable collision notices

No transcript duplicationNative Codex remains the only transcript authority

What left the core

No resident Coordinator, background monitor, or self-repair engine.

SessionStart reads only a small opt-in marker and launches nothing. Doctor is read-only and tells you to update or reinstall. Pull requests are optional. Provider, schedule, release, database, and deployment actions remain with the task and user who own them.

Runtime

Zero third-party runtime dependencies

Requires Codex, Git, and an existing Python 3.10+ interpreter. No database, daemon, queue, pip package, npm package, Python installer, or OS package-manager call.
