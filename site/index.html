The problem arrives before the tool

One job becomes four task windows. Then the mess starts.

!

This probably sounds familiar

  • Two agents investigate the same problem and give you two different fixes.
  • One agent changes a file while another is still relying on the old version.
  • A task pauses or restarts, and nobody is sure what was finished.
  • You read every task window just to learn whether the project is actually done.
✓

You do not need it for everything

  • One agent can finish the job safely on its own.
  • You only need a quick answer or one small edit.
  • Short-lived helpers can report straight back to one parent task.
  • Your work is already cleanly separated and the handoff is obvious.

What changes for you

You describe the finish line. You stop carrying every handoff.

01 / The split

You stop assigning every little step

Coordinator turns the outcome into a few clear work areas and keeps each area with the same agent through the fix, tests, and follow-up.

02 / The limit

Five, not fifty

It caps the default team at five active workers. More task windows are not automatically more progress.

03 / The restart

A pause does not erase the plan

If a task pauses, compacts, or restarts, the next turn can recover who was doing what and what still needs attention.

04 / The updates

You are no longer the message bus

Agents keep their findings with their own work. Coordinator gathers the useful parts and gives you one project update instead of constant status chatter.

05 / The review

Review waits for finished work

The reviewer checks one stable result instead of chasing files while other agents are still changing them.

06 / Your say

Agents still cannot overrule you

Coordination does not turn a suggestion into permission. Publishing, deployment, database changes, and other important decisions still wait for your approval.

07 / Optional view

Mission Control is optional

The core Coordinator works without a dashboard. When started locally, Mission Control separates queued work from work actually running, shows native task names and next actions, and keeps project state on your machine.

08 / Small by design

Zero third-party runtime dependencies

It uses Python's standard library—no orchestration framework, database, queue, pip package, or npm package to install.

Direct answers

Questions developers ask before adding another coordination layer.

What is Codex Coordinator?

A free, open-source plugin that coordinates multiple durable OpenAI Codex tasks in one Git repository and returns one checked project update.

Can Codex run multiple agents without it?

Yes. Coordinator is useful when separate durable tasks need explicit ownership, recoverable handoffs, consolidated status, and one project-level result.

Does it replace Git worktrees?

No. Worktrees isolate branches and files. Coordinator tracks who owns each job, what is blocked, what survived a restart, and what still needs review.

Do I need Ultra?

No. You can explicitly ask Coordinator to delegate work. Normal Codex plan, model, usage, token, and concurrency limits still apply.

Does it send code to a coordination server?

No separate coordination service is added. It uses native Codex tasks and small local project records; its restart helper makes no network calls.

When should I avoid it?

Use one Codex task for a small change and parent-owned subagents for short help. Coordinator is for several durable work areas where ownership and recovery matter.
