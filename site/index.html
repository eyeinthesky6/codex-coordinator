Goal-scoped coordination, not background orchestration

Keep parallel Codex tasks from crossing wires.

Ask one Coordinator to reuse related local tasks before assigning two or three complete work lanes in your current checkout. Each task publishes only its active goal and planned boundary. No polling, extra worktrees, or transcript mirror.

What left the core

No always-on Coordinator, background monitor, or self-repair engine.

A user-invoked Coordinator stays available for its goal and reads the active board when asked; it never polls workers. SessionStart launches nothing. Doctor is read-only. Tasks cooperatively commit reviewed exact files on the shared branch, and pull requests remain optional.

Runtime

Zero third-party runtime dependencies

Requires Codex, Git, and an existing Python 3.10+ interpreter. No database, daemon, queue, pip package, npm package, Python installer, or OS package-manager call.
