# Governance

Codex Coordinator is currently a maintainer-led project.

## Current roles

- **Maintainer:** Six Ideas (`@eyeinthesky6`) owns product direction, releases, security coordination, repository access, and final decisions.
- **Contributors:** anyone may report a problem, improve documentation, add a reproduction or test, or propose a focused change through the documented public routes.

No additional maintainer or privileged community role is currently open. The project should add roles only when a real support or review queue requires them.

## How decisions are made

- Reversible implementation details are decided in pull requests using code, tests, and documented behavior as evidence.
- Usage questions and early ideas start in GitHub Discussions so people can add examples before implementation is scoped.
- Broad protocol, compatibility, architecture, security, release, access, licensing, and conduct decisions remain maintainer-owned.
- Comments and reactions are useful evidence, but do not automatically set roadmap priority.

Accepted work needs a clear user problem, bounded scope, explicit non-goals, and a validation path. The maintainer may decline changes that duplicate native Codex behavior, add project-specific policy, expand the trusted runtime without clear benefit, or exceed sustainable support capacity.

## Growing project roles

If the project develops a real need for help, access should grow gradually: community helper, contributor, bounded reviewer or triager, then maintainer. Decisions will consider sustained useful work, reliability, judgment, respectful conduct, privacy and security awareness, candidate consent, and the minimum permissions needed. Activity count alone is not enough.

The maintainer approves role and access changes. Inactivity or a changed project need may end a role after the change is communicated and work is handed off safely.

## Review condition

Review this model when support volume, contributor activity, release duties, or security ownership can no longer be handled reliably by one maintainer.
